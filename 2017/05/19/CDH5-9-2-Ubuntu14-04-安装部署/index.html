CDH5.9.2 Ubuntu14.04 安装部署

  • 大数据




因为毕设需要用到 Hadoop 生态环境中的很多组件，一个个安装和维护十分不方便，并且没有直观的页面显示，所以采用 Cloudera 公司的 CDH（Cloudera’s Distribution Including Apache Hadoop），这是中国绝大部分公司使用的。

这篇文章主要介绍 CDH 在 Ubuntu 下的安装与部署。刚开始采用的是在线安装的方式，到最后下载 Parcels 的时候实在是太慢，如果以后要添加节点必然也是十分耗时的，所以换成了离线安装方式，即把所需的包事先下载好，然后执行操作指令即可。这里所需要的资源有一下5个：

  • CDH-5.9.2-1.cdh5.9.2.p0.3-trusty.parcel
  • CDH-5.9.2-1.cdh5.9.2.p0.3-trusty.parcel.sha1
  • manifest.json
  • cloudera-manager-trusty-cm5.9.2_amd64.tar.gz
  • jdk-8u121-linux-x64.tar.gz

也可以自行参考官方文档下载，也可以用文末提供的下载链接，用百度云下载（因为从官网下 cm 的时候是异常的慢，我是通过在学校走 IPV6下载的）。

资源准备好后就开始主机的配置了，我这里搭建的是一个很小的集群，一台 master三台 slave（slave01，slave02，slave03）配置如下：

Node Memory Storage
Master 16G 1T
Slave01 8G 500G
Slave02 8G 500G
Slave03 4G 500G

这里假设已经初步安装好了 Ubuntu14.04（不要安16.04，至少写这篇文章的时候是CDH 是不支持16.04的）

使用初始脚本配置数据库scm_prepare_database.sh（主节点）

这个脚本就是用来创建和配置CMS需要的数据库的脚本。各参数是指：

  • mysql：数据库用的是mysql，如果安装过程中用的oracle，那么该参数就应该改为oracle。
  • -uroot：root身份运行mysql。-proot：mysql的root密码是root。
  • –scm-host localhost ：CMS的主机，一般是和mysql安装的主机是在同一个主机上。
  • 最后三个参数是：数据库名，数据库用户名，数据库密码。

下载相关的依赖（这个我是参考其它博客，我认为 CDH 应该会自己下载，但是可能会很慢？）:

至此基本的集群搭建好了。可以运行一个 mapreduce 任务看下效果：

然后我们在终端启动下各个服务看看效果，比如 hive，spark这里可能会报AccessControlException: Permission denied: user=root, access=WRITE, inode=”/user”:hdfs:supergroup:drwxr-xr-x错误。

这是因为root去访问hdfs上面的/user目录没有权限。所以最后，我在主节点上面先使用命令 su hdfs切换到hdfs用户，然后通过Hadoop fs -chmod 777 /user 来进行授权。

运行 Spark-shell 还可能会报：Required executor memory (1024+384 MB) is above the max threshold (1057 MB) of this cluster! Please check the values of ‘yarn.scheduler.maximum-allocation-mb’ and/or ‘yarn.nodemanager.resource.memory-mb’.错误。

如果没有 CDH 的话我们可能就要自己去找配置文件，然后改了，这里直接在网页端操作就好了：在 yarn 管理页面找到相应的两个文件然后修改，将executor memory改大，然后就可以打开了。

这里还要说明下我犯的一个低级错误，远程访问 CDH，在 CDH内打开各个服务的 webui 时直接访问是打不开的，因为它是根据主机名而不是 IP 去访问的，而我们所使用的笔记本上可能没有映射，自然就访问不到了，所以最好是将主机名改为 IP 就好了。
